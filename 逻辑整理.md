A. 活动方案确立


1. 我方手动完成初稿 - 初稿格式化输出 - 人工复核（比较简单）
我希望团队材料提供：手动初稿 - 包括客户（或组织）名称，具体日期和时间，人数，参观行业意向（新能源， 电子制造， 无人机， AI大模型），特殊要求（宗教，饮食），所需陪同人数，

质量控制，让2-3个大语言模型进行分析和输出相同格式的内容：

关键字段以及AI输出格式：

客户：char
日期：TIME
人数：json
所需陪同人数：json
参观行业意向：
json：{
    1. xxxx，基于公司已有数据/经验的大致公司选择 （此处因为是人工初版，所以使用公司数据，不用AI推荐数据）
    2. xxxx，基于公司已有数据/经验的大致公司选择
    3. xxxx，基于公司已有数据/经验的大致公司选择
}
特殊要求：json
其他信息：
json: 
{
    1.
    2.
}

质量控制：在输出的prompt中，让AI从初稿中挑选他认为需要特别向合作方澄清的信息，避免miscommunication, 比如人数是否一直一致，参观行业的优先级权重，特殊要求，专门拉一个字段来进行存储，并且加上checkbox，在后续会议中进行询问澄清，并且记录结果。

将2-3版的AI输出内容进行比对，查看不同的地方进行人工核验。对于需要修改的地方，进行人工复核修改。

输出可以被导出为excel/csv/txt格式，且有中英双语版本。


2. 与合作方线上碰头讨论初稿方案

会议中应：
确认基本信息（日期，人数）；跟客户确认参观意向，根据客户反馈来继续完善参观或会议需求，可记录具体/模糊需求。
具体需求 和 模糊需求分开记录（最好）。

AI会议笔进行会议总结/人工会议记录。

3. 基于会议纪要，补充初稿活动信息 
  - 会议纪要结合初稿，进行格式化输出
  - 和初稿的主要区别在于参观行业/会议的需求的变化，是否有增改。
  分为：
  精确需求（json）
  模糊需求（json）：需要AI帮忙查看用户暗藏的意思 - 寻找与需求相关的行业/公司 - 列出一些选项（比如，10个）。让2-3个大语言模型互相监督和审核，最后挑出5个他们都认可的「由模糊转变为精确」的需求 （质量控制）
  
  待人工复核后（需要人工查看AI对于需求的分析是否合理），放弃/加入到精确需求列表中。

  最后的输出，字段格式应该跟初稿相同。工具应该给出初稿和会议后定稿的视觉比对，由负责同事进行人工确认。

输出可以导出为csv/excel/word/json格式，最后，生成一份PFD/WORD格式的合作方brief。



## 评估方案（可量化指标，用于持续改进）

1. 人工编辑时间
   - 记录每次人工编辑与复核的用时（分钟）：从AI输出结构化初稿开始，到负责人确认并导出可对外版本结束。
   - 用于衡量整体效率提升，观察趋势变化。

2. 来回次数（version_id + 关键字段改动）
   - 每次导出/对外发送都记录一个 `version_id`（例如v1/v2/v3），用于统计从初稿到定稿的版本轮次。
   - 同时记录关键字段的改动次数与改动点（人数/日期/特殊要求/陪同人数等），用于评估用户习惯与信息稳定性（是否频繁更改关键行程信息），并反向指导澄清环节与规则校验的优化。

3. 首次通过率
   - 用于评估Prompt与模型效果：定义为初稿或会议后定稿在首次提交评审时，无需补充关键信息、无需事实性纠错即可通过的比例。
   - 若通过率长期走低，需要迭代Prompt（澄清问题、结构化输出规则、事实/不确定标注），必要时考虑更换或组合模型。

并用AI根据会议纪要+优先级+公司业务模版，生成一份cold reach out的email或是电话沟通话术，进行沟通。沟通后，在每个沟通成功的企业/组织后面进行备注（是否同意，可行时间，费用，其他要求）
