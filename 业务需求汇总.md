### 复制给 Cursor 的开发文档 (Agent Workflow PRD)

**Project Name:** AI Partner Brief Generator (跨境政企访学专属版)
**Tech Stack Suggestion:** Next.js (App Router), TailwindCSS, shadcn/ui (for clean, professional UI), OpenAI API / Vercel AI SDK (for LLM calls).

#### 1. 业务背景 (Business Context for AI)
这是一个用于“高端跨境政企访学”的内部效率工具。目标客户群（如中东政要/高管）通常需求模糊（例：“想看中国顶尖 AI”）。对接公司的痛点是：需要将这些模糊需求转化为具体的、可落地的、且成本可控的国内参观方案。
**核心破局点：** Agent 不能只是个打字员，必须具备**商业意图洞察**和**平替资源匹配 (Alternative Routing)** 的能力。

#### 2. 系统核心工作流 (The 4-Step Pipeline)
请在后端 API 或 Server Actions 中实现以下 Pipeline 串联逻辑：

* **Step 1: 需求解析与结构化 (Intent Parsing)**
    * **Input:** 业务员输入的非结构化文本（客户背景、原始聊天记录、初步意向）。
    * **Action:** 调用 LLM，提取 JSON 格式的关键字段：`client_profile`, `core_intent`, `hard_constraints`。
* **Step 2: 初稿生成与平替推荐 (Drafting & Alternative Routing) - 【核心亮点】**
    * **Input:** Step 1 的 JSON 数据。
    * **Action:** 调用 LLM。除了生成常规的参观目标，**必须**基于客户诉求推演并推荐 1-2 个“降级/平替方案”。（例如：想看小米人形机器人 -> 推荐参观扫地机器人产线或大疆代理商，以降低对接难度）。
* **Step 3: 会议纪要融合 (Meeting Integration)**
    * **Input:** 合作方碰头会的语音转录文本 (Transcript) + Step 2 的初稿。
    * **Action:** LLM 执行 Diff 对比，补充遗漏信息，更新 TBD（待定）事项，生成更新版的 Brief。
* **Step 4: 最终校验与输出 (Quality Control & Final Output)**
    * **Action:** 独立的 Reviewer LLM 节点。检查 Step 3 的输出是否包含“幻觉”（未提及的承诺），并确保所有未确认的金额/时间强制使用 `[TBD]` 占位符标识。温度 (Temperature) 设为 0.1。

#### 3. 前端 UI 布局设计 (Frontend UI/UX)
为了在面试投屏时效果最好，建议采用**左侧输入/控制台 + 右侧实时预览 (Split-pane view)** 的设计：
* **左侧面板 (Control Panel):**
    * **Phase 1 选项卡:** 一个 Textarea 用于输入“原始客户需求”，一个 Textarea 用于输入“客户背景”。点击“Generate Initial Brief”触发 Step 1 & 2。
    * **Phase 2 选项卡:** 一个 Textarea 用于输入“线上碰头会纪要”。点击“Integrate Meeting Notes”触发 Step 3 & 4。
* **右侧面板 (Preview & Output):**
    * **可视化 Workflow 进度条:** 用简单的 UI 标明当前跑到哪个节点了（解析中 -> 生成中 -> 校验中）。
    * **Markdown 渲染区:** 实时展示生成的 Brief。
    * **Badge/高亮:** 将生成的“备选/平替方案”用显眼的颜色（如琥珀色或蓝色）Highlight 出来，凸显 Agent 的商业价值。

#### 4. 核心系统提示词 (System Prompts for Cursor to use)

**Prompt A: 平替方案生成器 (用于 Step 2)**
```text
你是一个资深的跨境商务拓展专家。你的任务是根据客户的模糊需求，制定初步的接待 Brief。
最关键的是，你需要提供“平替/备选方案 (Alternative Routing)”。
如果客户提出了极高门槛的参观诉求（如指定看某头部大厂的核心机密业务），请结合其真实意图，推荐 1-2 个对接门槛更低、成本更低，但同样能满足其考察目的的国内企业或行业。
输出必须包含以下结构：
1. 客户核心意图洞察
2. 理想参观标的
3. 备选/平替参观方案及推荐理由 (Highlight this)
```

**Prompt B: 质检与反幻觉 (用于 Step 4)**
```text
你是一个严格的合规与质量控制 Reviewer。请对比【原始输入材料】与【Draft Brief】。
规则：
1. 找出所有 Draft 中存在，但原始材料中未提及的“具体时间、具体金额、具体承诺”，并将其替换为红色标记的 [需在碰头会中澄清 (TBD)]。
2. 严禁基于常识自行脑补补全任何商务条款。
```

#### 5. 评估指标展示区 (Dashboard / Metrics)
在页面底部或一个单独的 Modal 中，静态写死几个 Dashboard 指标，用于回应面试题的“评估方案”要求：
* **Avg. Editing Time:** 45 mins ➔ 8 mins (下降 82%)
* **First-Pass Acceptance Rate:** 92% (首次通过率)
* **Alternative Adoption Rate:** 35% (平替方案采纳率 - 证明节省了对接成本)
